A composite GitHub Action and small Linux eBPF agent built for GitHub-hosted Ubuntu runners. Record process and network signals in detect mode, or optionally enforce an egress allowlist when you are ready to harden the boundary.

Built for merge gates you can reason about

ColdStep is intentionally narrow: Linux-only, validated on ubuntu-latest, with telemetry written where you already look—workspace files and the Actions UI. Prefer JSONL when you need a durable trail; use the digest when you want a fast human read.

Kernel-adjacent truth

Syscall-class visibility for TCP connects, DNS, optional TLS ClientHello SNI, filesystem signals, and process relationships—without pretending the runner is a perfect forensic appliance.

Progressive hardening

Start in detect mode to learn normal egress. When policy is clear, switch to enforce mode with explicit allowlists so unexpected destinations fail the job early instead of silently leaking.

CI-native outputs

JSONL stream, shutdown telemetry JSON, and a Markdown digest you can merge into the job Summary—so reviewers see the same story you do in the PR thread.

Two modes, one agent

Pin a release tag in workflows (for example coldstep-io/coldstep@v0.1.6) instead of tracking @main unless you intentionally want churn.

Detect

Observe and record. No egress blocking. Ideal for baselining and incident-friendly timelines.

Default

Enforce

Block TCP and UDP egress outside your allowlist. Requires explicit configuration—this is a guardrail, not a surprise.

Opt-in

Artifacts you can diff

Paths default under $GITHUB_WORKSPACE and can be overridden with environment variables such as COLDSTEP_EVENTS_LOG when you need a stable contract for downstream tooling.

  • .coldstep-events.jsonl Append-only event stream—treat this as the source of truth for investigations.
  • .coldstep-detect.md Shutdown digest with triage ribbon, KPI tables, and collapsible sections for quick scans.
  • .coldstep-telemetry.json Totals and BPF health at shutdown for operational dashboards.

Optional AlienVault OTX enrichment

For detect workflows that build the bundled HTML report and job Summary, you can layer AlienVault Open Threat Exchange (OTX) lookups on IPv4 and hostname indicators. Results feed verdict columns and charts in the Markdown summary and self-contained HTML artifact—without changing core egress telemetry.

  • OTX_API_KEY Store your key as a repository or organization secret with this name. Create a free OTX account at otx.alienvault.com if you need a key.
  • If unset Enrichment is skipped entirely: no calls to OTX, no API usage, and detect jobs still finish successfully. Fork PRs and repos without the secret behave the same way.
  • Best-effort The enrichment step does not fail the job on third-party errors; rate limits and budgets surface as warnings or partial results in the report model. See the pipeline description in detect report tooling for environment variables and schema details.
